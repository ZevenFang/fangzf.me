> 小米 MiMo Orbit 百万亿 Token 创造者激励计划已经开放申请：30 天内免费发放总计 100 万亿 Token 权益，面向个人开发者、团队和企业。本文整理申请入口、适合人群、评估要点和使用场景。


![封面](/images/wechat-sync/小米突然发-100-万亿-Token-开发者现在就能申请/84f0bb929bb08ea0.webp)

![Xiaomi MiMo 100T Token 计划](/images/mimo-100t-cover.png)

4 月 28 日，小米技术官方宣布启动 Xiaomi MiMo Orbit 百万亿 Token 创造者激励计划：将在 30 天内免费发放总计 100 万亿 Token 权益，并同步推进 Agent 生态共建计划。

如果你正在做 AI 应用、Agent、Coding 工具链、自动化工作流，或者只是想低成本试用 MiMo 系列模型，这次活动值得看一眼。

申请入口：

https://100t.xiaomimimo.com/

## 一、这次活动是什么？

Xiaomi MiMo Orbit 百万亿 Token 创造者激励计划，是小米 MiMo 面向全球 AI 创作者、开发者和团队的一次限时 Token 定向发放活动。

按照活动页说明，小米计划在 30 天内发放总计 100 万亿 Token 权益，完全免费。活动不是“人人自动到账”，而是申请制：提交表单后，官方会根据项目情况、使用场景和证明材料进行评估。

活动时间为北京时间 2026 年 4 月 28 日 00:00 至 5 月 28 日 00:00，权益赠完即止。

## 二、谁适合申请？

从活动说明看，它并不只面向大公司。个人开发者、团队、企业都可以参与，关键在于你是否有清晰的 AI 使用场景。

更适合申请的人包括：

- 正在开发 AI Agent、自动化助手、Coding Agent、数据分析工具的开发者
- 正在把 Claude Code、Cursor、OpenClaw 等工具接入不同模型后端的用户
- 有明确产品原型、开源项目、内部工具或业务场景的团队
- 想测试 MiMo 系列模型在推理、规划、工具调用、多模态理解等场景表现的人
- API 消耗量较高，希望先用免费权益完成验证的 Builder

如果只是随手试用，当然也可以提交，但评估通过率和权益档位大概率取决于你能否讲清楚“准备用这些 Token 做什么”。

## 三、能拿到什么权益？

活动页写得比较明确：官方会根据表单内容和开放平台账号状态，发放不同额度、不同档位的 Token Plan，或不同金额的赠金。

Token Plan 是开放平台的订阅制套餐，可直接配置到 Claude Code、Cursor 等编程工具中使用 MiMo 系列模型；赠金则会进入开放平台账户余额，用于按量计费的 API 调用。

具体额度以最终到账为准。

## 四、怎么申请？

申请流程并不复杂：

### 1. 打开申请页：https://100t.xiaomimimo.com/
### 2. 点击「立即申请」，填写并提交表单
### 3. 等待评估，活动页提示大约 3 个工作日
### 4. 评估通过后，查看通知邮件
### 5. 使用表单填写的邮箱登录或注册 Xiaomi MiMo API 开放平台
### 6. 权益通常会在收到邮件后的 24 小时内到账

这里有一个容易忽略的点：邮箱要对上。

如果你已经有开放平台账号，请确认账号由表单填写的邮箱注册；如果账号是手机号注册，需要先绑定该邮箱。否则即使评估通过，也可能影响权益到账。

## 五、申请表怎么填更稳？

这类资源申请，本质上不是拼口号，而是让评估方判断：你是否真的会使用这些 Token，以及使用场景是否足够具体。

建议填写时重点说明：

- 你正在做什么项目，最好有项目名称、链接、截图或 Demo
- 你准备用 MiMo 处理哪类任务，例如代码生成、工具调用、文档理解、客服 Agent、数据分析、多模态理解等
- 你当前使用哪些 AI 工具或模型，以及为什么想测试 MiMo
- 预计调用规模和验证目标，例如 Benchmark、PoC、插件适配、真实业务试点
- 如果是开源项目，附上 GitHub、文档页、产品页或社区链接

表述越具体，越容易让人判断你的需求是真实存在的。

## 六、MiMo-V2.5 为什么值得关注？

这次活动并不是孤立的补贴动作。参考公开信息，小米同时宣布 Xiaomi MiMo-V2.5-Pro 正式开源，并完成了与 SGLang、vLLM 等主流推理框架的 Day 0 适配。

MiMo-V2.5 系列面向复杂任务场景，重点覆盖 Agent 与 Coding 应用。官方信息中提到，MiMo-V2.5-Pro 在 GDPVal-AA、ClawEval 等榜单上表现突出；MiMo-V2.5 则是原生全模态模型，支持文本、图像、视频和音频理解。

这意味着它并不只是一个聊天模型，而是在往“可接入工具、可承担复杂任务、可进入开发者工作流”的方向推进。

## 七、对开发者的实际意义

对开发者来说，100T 计划最现实的价值有三个。

### 1. 降低试错成本

很多 Agent 或 Coding 场景一旦进入真实测试，Token 消耗会非常快。免费权益可以让团队先完成一轮集成、评测和成本估算。

### 2. 多一个国产模型后端选择

如果你已经在使用 Claude Code、Cursor、OpenClaw 等工具，MiMo API 开放平台提供了新的模型后端选择。对需要多模型路由、成本控制或国产化适配的团队，这是一个可以纳入测试池的选项。

### 3. 推动 Agent 生态适配

小米同时提出 Agent 生态共建计划，并已与 OpenCode、Hermes Agent、KiloCode 等 Agent 框架厂商展开合作。对框架作者和工具开发者来说，这类合作通常意味着更快的适配、更直接的反馈通道，以及更完整的模型能力验证机会。

## 八、申请前的检查清单

提交前可以快速确认这几项：

- 邮箱是否正确，并能正常收信
- 开放平台账号是否已用同一邮箱注册或绑定
- 项目描述是否具体，不只是“想体验 AI”
- 是否准备了证明材料，例如项目链接、截图、Demo、GitHub 仓库
- 是否知道权益有有效期，到账后要及时使用

如果 3 个工作日内没有收到评估通过邮件，可以检查垃圾邮件箱，确认邮箱是否填写正确；仍未收到时，可以尝试重新提交表单，或通过开放平台「联系我们」反馈。

## 九、最后

小米这次把 MiMo-V2.5 开源、API 平台、Token 激励和 Agent 生态共建放在同一时间窗口推进，信号很明确：它想进入开发者真实工作流，而不只是做模型发布。

如果你正在做 AI 产品、Agent 框架、Coding 工具链或企业内部自动化，现在申请这批免费 Token，至少能换来一次低成本实测机会。

申请入口：

https://100t.xiaomimimo.com/
